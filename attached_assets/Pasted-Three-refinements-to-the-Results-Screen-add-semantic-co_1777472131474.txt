Three refinements to the Results Screen: add semantic colors throughout, vary the stat cards' identity colors, and add a discreet "mark as difficult" feature on missed words.

PART 1 — Semantic colors on the weekly chart
Currently the line chart uses a uniform amber gradient. Replace this with performance-based color coding:
Each data point dot, and the line segment leading to/from each point, should be colored based on the score for that day:

score >= 80 → green #10B981 (excellent)
score 50-79 → yellow #FBBF24 (medium)
score < 50 → red #EF4444 (low)

Implementation:

Render the line as multiple SVG segments, each colored according to the average of its two endpoint scores
Each data point dot uses its own color based on its individual score
The area below the line (the gradient fill) should be a soft amber #D97706 at 12% opacity, fading to 0% at the bottom — keep this amber to maintain brand cohesion (don't multi-color the fill, only the line and dots)
The tooltip on hover should also reflect the score color (e.g., "Tue — Score 80%" with the percentage in green if >=80)

Maintain smooth bezier curves between points. The transition from one color segment to another should be a hard-edge transition at the data point itself (cleaner than a gradient blend, makes the performance level immediately readable).
